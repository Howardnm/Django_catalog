# 产品电子手册 (Catalog) App 实施计划

## 1. 业务目标
为外部客户提供一个专业、直观的产品电子手册展示页面，支持按分类筛选、详细参数查看及技术文档（TDS/MSDS/RoHS）下载。该模块后期将支持独立运行，并与主系统进行数据同步。

## 2. 核心功能
- **分类导航**：按塑料类型（通用、工程、特种等）和应用场景进行分类。
- **产品搜索**：支持牌号、厂家、特性等关键字搜索。
- **详情展示**：展示材料的各项物理、机械性能指标。
- **文档下载**：安全下载对应的技术文档。
- **架构解耦准备**：
    - 设计 API 接口层，用于后期外部系统调用。
    - 数据库模型设计考虑跨系统引用。

## 3. 技术构思 (关于解耦运行)
为了实现后期迁移和安全隔离，我们将采取以下策略：
1. **数据同步方案**：采用 **API + Webhook** 模式。主系统（app_material）作为数据源，当物料更新时通过 Webhook 通知手册系统；手册系统通过 API 定时同步或按需拉取。
2. **文件存储**：建议使用对象存储 (如 MinIO/OSS/S3)，这样两个系统可以共享同一个存储地址，或者由主系统提供带签名的下载链接。
3. **认证鉴权**：手册系统将拥有独立的客户访问权限体系，不直接共享主系统的 User 表。

## 4. 实施路线图 (第一阶段)
1.  **模型设计** (`models.py`)：
    -   `CatalogCategory`：手册特有的分类体系（可映射自 `MaterialType`）。
    -   `CatalogProduct`：手册展示的产品镜像，包含性能快照和文档链接。
2.  **API 开发**：
    -   在 `app_material` 中增加导出 API。
    -   在 `app_catalog` 中增加同步接收逻辑。
3.  **前端页面**：
    -   首页：分类概览。
    -   列表页：过滤与搜索。
    -   详情页：性能图表化展示。
